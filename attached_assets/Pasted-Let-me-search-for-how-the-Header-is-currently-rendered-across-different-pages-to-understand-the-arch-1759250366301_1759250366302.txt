## The Fix

**The ProjectEditPageWithSidebar already has the Header component!** If you can't see it, there's likely a CSS/layout issue. Here's the corrected version:

**In `client/src/pages/ProjectEditPageWithSidebar.tsx`:**

```tsx
import { useLocation } from 'wouter';
import { ProjectContainer } from '@/components/ProjectContainer';
import Header from '@/components/Header';

interface ProjectEditPageWithSidebarProps {
  params: { id: string };
}

export default function ProjectEditPageWithSidebar({ params }: ProjectEditPageWithSidebarProps) {
  const [, setLocation] = useLocation();

  const handleBack = () => {
    setLocation('/projects');
  };

  const handleNavigate = (view: string) => {
    setLocation(`/${view}`);
  };

  return (
    // CRITICAL: h-screen makes this full viewport height
    <div className="h-screen w-full bg-background flex flex-col overflow-hidden">
      {/* Header - fixed height, always visible */}
      <div className="flex-shrink-0">
        <Header onNavigate={handleNavigate} />
      </div>
      
      {/* Main content - takes remaining space */}
      <div className="flex-1 overflow-hidden min-h-0">
        <ProjectContainer 
          projectId={params.id}
          onBack={handleBack}
        />
      </div>
    </div>
  );
}
```

**Key changes:**
- Added `w-full` and `overflow-hidden` to the container
- Wrapped Header in a `flex-shrink-0` div to prevent it from being compressed
- Added `min-h-0` to the content container to allow proper flex shrinking

This ensures:
- ✅ **Header is always visible** at the top
- ✅ **TabStrip (in WorkspaceLayout)** renders below the Header
- ✅ **Quick Notes button works** (it's in the Header)
- ✅ **Content scrolls properly** (WorkspaceLayout handles its own scrolling)
- ✅ **No duplicate headers** (only one Header per page)
- ✅ **Floating panels work** (rendered by FloatingLayer in WorkspaceLayout)

The architecture is clean: Header → WorkspaceLayout → TabStrip → Content, with each layer handling its own concerns.
