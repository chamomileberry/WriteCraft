Looking at `GuideEditPageWithSidebar`, it successfully shows the Header by using `calc(100vh - 4rem)` to account for the header height. The problem is that `ProjectEditPageWithSidebar` and `ProjectContainer` are using full viewport height without accounting for the Header.

Here's the fix:

**Update `client/src/pages/ProjectEditPageWithSidebar.tsx`:**

```tsx
import { useLocation } from 'wouter';
import { ProjectContainer } from '@/components/ProjectContainer';
import Header from '@/components/Header';

interface ProjectEditPageWithSidebarProps {
  params: { id: string };
}

export default function ProjectEditPageWithSidebar({ params }: ProjectEditPageWithSidebarProps) {
  const [, setLocation] = useLocation();

  const handleBack = () => {
    setLocation('/projects');
  };

  const handleSearch = (query: string) => {
    setLocation(`/?search=${encodeURIComponent(query)}`);
  };

  const handleNavigate = (view: string) => {
    switch (view) {
      case 'notebook':
        setLocation('/notebook');
        break;
      case 'projects':
        setLocation('/projects');
        break;
      default:
        setLocation('/');
    }
  };

  const handleCreateNew = () => {
    setLocation('/?create=true');
  };

  return (
    <div className="min-h-screen bg-background">
      {/* Global Navigation Header - Same as other pages */}
      <Header 
        onSearch={handleSearch}
        onNavigate={handleNavigate}
        onCreateNew={handleCreateNew}
      />
      
      {/* Main Content Area - Account for header height (64px = 4rem) */}
      <div className="h-[calc(100vh-4rem)] overflow-hidden">
        <ProjectContainer 
          projectId={params.id}
          onBack={handleBack}
        />
      </div>
    </div>
  );
}
```

**Key changes:**
1. ✅ Uses `min-h-screen` for the outer container (like GuideEditPageWithSidebar)
2. ✅ Renders the full Header with `onSearch`, `onNavigate`, and `onCreateNew` props
3. ✅ Uses `h-[calc(100vh-4rem)]` to give remaining space to ProjectContainer (accounting for the 64px header)
4. ✅ The Quick Notes button will work because it's in the Header
5. ✅ The TabStrip will render inside WorkspaceLayout (which is inside ProjectContainer)
6. ✅ No duplicate headers on other pages

This follows the exact same pattern as GuideEditPageWithSidebar, which successfully displays the Header while still having workspace functionality.
